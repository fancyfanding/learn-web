# 简历投递策略

投递时间：4月份左右，下半年7到10月左右高峰期

时间：一般周一到周五都投递简历，特别是周三，每天早上8:45投一下，中午**13:30-15:30**投递，晚上22-23点投递，周六周日白天不投，没效果，每次投二三十家

渠道：

1. boss，智联招聘，前程无忧，拉勾招聘（主要是技术岗位），牛客，脉脉，猎聘，都装好，搞个文件夹
2. 内推成功率很高，能内推尽量内推
3. 不要轻易去国外
4. 企查查，公司注册时间、业务怎么样、是否暴雷，是否有风险
5. 去企业官网查询岗位，进行网申
6. 微信群，小红书，知乎微博招聘信息，相对较差
7. github开源社群，混脸熟，转化时间长

策略：优先招聘软件，其次找相关企业官网，开源社区可常活跃

# 心天地项目亮点

1. 简历中要写核心优势
2. 口语化讲项目亮点
3. S：项目背景，描述下项目是什么，解决什么业务需求；
4. T：自己的角色是什么，是否从0到1，是不是leader
5. A（核心内容）：行动，技术难点是什么，踩过哪些坑，用了哪些技术，为什么选用这个方案，如果重构做什么优化
6. R：行动过程中有哪些成果，获得的指标，上线的效果，优化的效果，eg.首屏加载时间的优化，包体积的压缩率……

## 以心天地为例：

> **凸显技术能力，将面试从销售思维转向为技术的讲解**

S：面向学生的心理平台，可以完成心理测评、社区、资讯、树洞的中后台项目，后台完成数据的上传，

T：确定自己为项目主导人，包装一下，有一些同学跟着，一起做的，是leader，从0到1做的，交付的内容就是PC端的管理闭环，小程序端的用户闭环

A：

1. 搭建了一个双端的前端架构：小程序端承载了社区内容，测评的流程，PC端承载的是运营管理能力，通过统一的数据模型，打通列表、详情、后台管理的数据链路，降低了状态不一致的问题
2. 沉淀互动组件化的能力，封装了评论，分享等通用的组件，severs层包含数据请求、兜底、状态更新的能力，保证了乐观更新，失败回滚的交互体感，避免了重复操作导致的数据错误的问题，提高了开发效率
3. 性能优化能力，针对数据密集的场景，对后台的表格进行渲染性能的治理，分页、按需渲染、懒加载

R：形成了多端的闭环能力，沉淀了可复用的组件，为后续开发提供了基础，开发效率提升了30%
