# 考虑的是人怎么样，让人感受到你是一个可靠的人（抗压、加班意愿、文化契合度）一定要举一个过去的例子，行为场景题最后一点都是反思

# 题型1 为什么加入这家公司

雷点：不谈钱

1. 我能从这家公司收获什么：
2. 我能够给这家公司带来什么价值：

要点

1. 我认为公司的使命价值观是我认同的
2. 业务和产品线是什么，赛道、用户群体、商业模式是我喜欢的，与我过往的经验、项目有强烈的关联度，个人契合度比较高
3. 想加入优秀团队，与团队共同成长
4. 总结性的说一下

# 题型2 技术指标怎么来

1. 技术类的通过AB测试，监控看板，lighthouse
2. 非技术类的，主观感受

# 题型3 你与团队成员发生冲突的例子，如何处理（待落实）

不要去讲个人恩怨，吐槽同事，冲突的点一般选择技术方案的差异，技术栈的选择，库的选择等

1. 拉回到业务指标
2. 通过埋点、ab测试进行对比，证明技术方案更优，更有利于未来的技术特点
3. 让领导参与决策、担责
4. 反思（行为和技术力都行，避开会深挖的技术点）

# 题型4 如果说为了完成某个项目，不得不学习一个新的技能，结果如何

要讲学习方法，平时是如何学习的，展示适应度和主动性，学习方法和学习能力

1. 展示自己的学习方法，具体的行为，看官网、看源码、用AI
2. 反思，如何帮助公司减少团队的学习成本，整理文档分享

# 题型5 如果说背调了你的同事，让你的同事讲你的缺点，觉得会是什么

1. 过分的追求技术细节，对项目细节的把握上有点苛刻，会花一点时间去解决代码整洁度的问题
2. 如何改进这个缺点的，效率也是蛮重要的，效率和质量的平衡，根据项目的优先级分配精力
3. 不要说自己没有缺点，不能讲一些致命伤，（迟到、情绪不稳定、抗压能力差）


1. 在平时工作过程当中，不太擅长拒绝，领导和同事安排的任务都会去接，会将工作时间压缩的比较紧，工作比较满
2. 改进，统筹自己的任务看板，结合时间精力去分析自我的任务规划
