# 考虑的是人怎么样，让人感受到你是一个可靠的人（抗压、加班意愿、文化契合度）一定要举一个过去的例子，行为场景题最后一点都是反思

# 题型1 为什么加入这家公司

雷点：不谈钱

1. 我能从这家公司收获什么：
2. 我能够给这家公司带来什么价值：

要点

1. 我认为公司的使命价值观是我认同的
2. 业务和产品线是什么，赛道、用户群体、商业模式是我喜欢的，与我过往的经验、项目有强烈的关联度，个人契合度比较高
3. 想加入优秀团队，与团队共同成长
4. 总结性的说一下

1. 技术类的通过AB测试，监控看板，lighthouse
2. 非技术类的，主观感受

# 题型3 你与团队成员发生冲突的例子，如何处理（待落实）

不要去讲个人恩怨，吐槽同事，冲突的点一般选择技术方案的差异，技术栈的选择，库的选择等

1. 拉回到业务指标
2. 通过埋点、ab测试进行对比，证明技术方案更优，更有利于未来的技术特点
3. 让领导参与决策、担责
4. 反思（行为和技术力都行，避开会深挖的技术点）

> 在之前的电商小程序重构项目中，我和同事就状态管理方案产生了分歧：我主张用 Pinia 做技术栈升级，同事担心迁移成本高，坚持沿用旧的 Vuex。
>
> 我先和他对齐了项目 "按时上线 + 长期可维护" 的核心目标，没有陷入争论，而是做了两个方案的性能、维护成本对比，同时给出了分阶段迁移的落地方案，把单次工作量控制在 2 天内，不影响上线。
>
> 之后我把数据和方案同步给技术负责人，最终团队采用了 Pinia 方案，顺利完成了项目。这件事也让我意识到，技术方案要平衡短期成本和长期收益，后续我会提前和同事对齐需求，避免冲突。

# 题型4 如果说为了完成某个项目，不得不学习一个新的技能，结果如何

要讲学习方法，平时是如何学习的，展示适应度和主动性，学习方法和学习能力

1. 展示自己的学习方法，具体的行为，看官网、看源码、用AI
2. 反思，如何帮助公司减少团队的学习成本，整理文档分享

>
> 之前公司用 UniApp 做跨端项目，我只有原生 Vue 经验，必须 2 个月内上手完成核心功能。
>
> 我先通过官方文档 + Demo 搭建了知识体系，再边做项目边学，查源码、用 AI 解决跨端适配的坑，同时把所有踩坑点整理成了手册。
>
> 最终我 1 个月就独立完成了核心模块开发，上线后适配问题率低于 5%，还把手册分享给团队，做了技术分享，把团队学习周期从 2 个月缩短到 1 个月，降低了团队的学习成本。

# 题型5 如果说背调了你的同事，让你的同事讲你的缺点，觉得会是什么

1. 过分的追求技术细节，对项目细节的把握上有点苛刻，会花一点时间去解决代码整洁度的问题
2. 如何改进这个缺点的，效率也是蛮重要的，效率和质量的平衡，根据项目的优先级分配精力
3. 不要说自己没有缺点，不能讲一些致命伤，（迟到、情绪不稳定、抗压能力差）
4. 在平时工作过程当中，不太擅长拒绝，领导和同事安排的任务都会去接，会将工作时间压缩的比较紧，工作比较满
5. 改进，统筹自己的任务看板，结合时间精力去分析自我的任务规划

> 同事可能会说我两个可改进的点：
>
> 第一，我对技术细节比较苛刻，有时会为了代码整洁度做重构，虽然长期降低了维护成本，但偶尔会让紧急需求的交付节奏稍紧。现在我会严格区分项目优先级，紧急需求先保交付，优化放到技术债务清理中，平衡效率和质量。
>
> 第二，我不太擅长拒绝，会承接太多额外任务，导致自己工作节奏紧张。现在我会用任务看板梳理优先级，合理评估时间，和同事沟通交付节点，统筹好自己的工作，避免过度消耗。
