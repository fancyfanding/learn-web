# 项目亮点（整体的讲述过程控制在6分钟以内）

## 心天地

1. 项目亮点1-中后台数据密集型表格的性能治理：

   S：需要运营同学高频查看一些信息，如问卷完成率，需要支持筛选、排序，当数据量上来之后，页面会出现卡顿、滚动不流畅，筛选延迟等问题

   T：针对以上问题，为了保证数据量较大时，能够交互流畅，筛选能够更加迅速，页面不卡顿

   A：

   1. 定位瓶颈，利用chrome performance 发现是因为dom节点过多，筛选条件触发了全量的计算和渲染；
   2. 减少渲染的开销就是核心的卡点，对长列表进行按需加载，虚拟滚动，把dom的数量控制在一定的区间，做服务端的分页，减少单页渲染的规模，减少重复计算（**防抖，节流**）,在筛选的时候加入debounce，防止每次输入的时候触发请求
   3. 交互稳定性，加入loding的状态，错误的兜底(try catch)

   R：提升了系统稳定性、流畅性，当数据量上来之后也能够流畅运行（量化指标）
